# Smart Chat Manager

Расширение для SillyTavern, которое привносит настоящую эргономику управления чатами во фронтенд, где ее, честно говоря, никогда не было. Оно автоматически придумывает названия для ваших чатов, позволяет добавлять к ним теги, содержит библиотеку промптов и менеджер API с поддержкой нескольких профилей, а также полностью перерабатывает панель чатов, добавляя поиск, сортировку и встроенные значки (бейджи).

---

## Функции

### 1. Автоматическое наименование чатов
- После достижения настраиваемого порога (по умолчанию: 6 сообщений) и только один раз за чат, запрашивает у LLM название, включающее `{{char}}`.
- Опциональное окно подтверждения позволяет принять или отклонить предложенный вариант.
- Предложенное название очищается от лишних символов и применяется через официальную функцию ST `renameChat()`.

### 2. Система тегов
- **Вручную:** добавляйте и удаляйте теги для текущего чата в панели настроек расширения.
- **Кнопка Авто-тегирования (Auto-Tag):** запрашивает у LLM 3-5 коротких ключевых слов-тегов, разделенных запятыми.
- Теги хранятся в `extension_settings.SmartChatManager.tags` с привязкой к имени файла чата. Сами `.jsonl` файлы чатов никогда не изменяются.

### 3. Переработка интерфейса прошлых чатов (Past Chats)
Интегрировано в существующее всплывающее окно управления файлами чатов (`#shadow_select_chat_popup`):
- 🔎 Строка поиска по тегам (поддержка нескольких терминов, логика И / AND).
- 🔃 Выпадающий список сортировки: По дате последнего сообщения / По дате создания / По алфавиту.
- 🏷️ Цветные значки тегов под названием каждого чата.

### 4. Библиотека промптов (менеджер системных промптов)
- По умолчанию поставляются два встроенных промпта: **Default Naming** (Стандартное наименование) и **Default Tagging** (Стандартное тегирование).
- У каждого сохраненного промпта есть тип (naming / tagging), поэтому он отображается только в соответствующем выпадающем списке.
- Редактируйте, создавайте и удаляйте промпты в модальном редакторе.
- Встроенные промпты нельзя удалить, но их **можно сбросить** к исходным настройкам в любое время.
- Поддерживаемые макросы в системном промпте: `{{char}}`, `{{user}}`. **История чата (транскрипт)** отправляется автоматически как сообщение пользователя.

### 5. Менеджер API с несколькими профилями и ротацией
- Сохраняйте неограниченное количество OpenAI-совместимых профилей API. У каждого есть имя, URL, ключ и модель.
- Выбирайте активный профиль из выпадающего списка или включите ротацию (Rotate), чтобы циклически переключаться между ними при каждом вызове (отлично подходит для распределения нагрузки от наименования/тегирования между ключами или для обеспечения отказоустойчивости, если один из профилей упрется в лимиты).
- Два формата запросов:
  - **Chat Completion** — отправляет стандартный массив сообщений с `{role:"system"}` + `{role:"user"}` (в стиле Tavern).
  - **Text Completion** — объединенный в один текст промпт для устаревших эндпоинтов `/completions`.
- Существующие установки с одной конфигурацией автоматически переносятся в профиль при первой загрузке.

### 6. Локализация (EN / RU)
- Полные английские и русские словари.
- Переключатель в панели настроек; обновляет интерфейс в реальном времени без перезагрузки.
- Уведомления (Toasts) и модальное окно переименования также переведены.

## Установка

### Вариант А — через интерфейс "Install Extension" в SillyTavern
1. Загрузите эту папку в публичный Git-репозиторий.
2. В SillyTavern перейдите в **Extensions (Расширения) → Install extension (Установить расширение)** и вставьте URL-адрес репозитория.

## Обработка ошибок

- Ошибки LLM (проблемы с сетью, ошибки HTTP, некорректный ответ) перехватываются и выводятся в виде всплывающего уведомления (Toastr), содержащего исходное сообщение об ошибке.
- Расширение помечает каждый чат меткой «попытка названия» после первого раза (успешно или нет), поэтому оно не будет докучать вам при каждом следующем сообщении. Используйте кнопку **Suggest a Name Now** (Предложить название сейчас), чтобы повторить попытку вручную.
- Сохранение тегов использует встроенный в ST механизм отложенного сохранения настроек (debounced settings saver) — ваши теги сохраняются после перезагрузки, смены персонажа и переименования чата.
